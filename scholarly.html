Informatix

1. Introduction

1.1 Purpose

The purpose of this document is to present in detail the specifications of the web aplication Informatix, based on the requirements of the Course Project in Web Technologies. The document will offer information about the requirements this project meets and how where those implemented.

This product is meant to be of use for students and teachers that want to solve and propose problems related to computer science. Teachers can use the apllication to create classes end evaluate their students, based on assigned homeworks.

1.2 Product Scope

The product is a web application designed for high school students to learn programming through problems. The application allows registered users to create, submit, and approve programming problems. The problems are tagged according to their difficulty level and programming concepts, such as dynamic programming and backtracking. The application allows teachers to create classes and add students, who have existing accounts, using their email or username. Teachers can also assign homework by selecting problems from the existing collection or proposing new ones. Students can solve the assigned problems and submit their solutions for evaluation by the teacher. They can also leave comments to provide hints or rate the problems using a star rating system.

2. Overall Description

2.1 Product Functions

Informatix web application is supposed to offer the following functionalities:

  • Give users the opportunity to propose computer science problems
  • Give users the opportunity to solve computer science problems
  • Provide users with feedback for their submitted solutions
  • Provide users with statistics based on their overall performance on the application
  • Login and SignUp forms. Users can choose a role beetween student and teacher
  • Teachers can create classes made up of users on this application
  • Teachers can assign homeworks to their classes
  • Teachers can evaluate the submits of the students in their classes
  • Users can change their profile picture.

2.2 User Classes and Characteristics

The users can be divided into four main categories, which are as follows:

  1. Guest users:
    • Have access to pages but cannot submit problems.
  2. Registered users, who can either be students or professors:
    • Students
      • Can join classes.
      • Can complete homework assignments.
      • Can solve problems on their own.
    • Professors
      • Can create classes.
      • Can assign homework assignments.
      • Can review homework assignments.
  3. Admins:
    • Can accept problem suggestions.
    • Can generate statistics.

2.4 Usage Manual

  1. Overview

    Our platform is designed to help users solve programming problems through a community-driven approach. Upon accessing the platform, you will be welcomed by the main page, which contains links to almost every other page. The top menu includes buttons for Home, Problems, About, Login, and Sign-up, while the left bottom corner of the page has a Help button.

  2. Login/Register

    To create an account or login to an existing one, click on the Login/Sign-up button in the top menu. You will then be taken to a form to register/enter your account information. Once you are done filling out the form, the buttons in the top menu will be replaced by the Profile page.

  3. Problems Page

    To access the Problems page, click on the Problems button in the top menu. This page contains a list of all the problems available on the platform.

  4. Problem Page

    If you click on any problem from the Problems page, you will be taken to that specific problem page. Here, you can see the problem description, comments left by other users, a solution page where you can see your past solutions, and a coding area where you can write your solution.

  5. Your profile Page

    To access this page, click on the Profile button from the top menu. Here, you can see your profile picture and you have the opportunity to change it at any time. Also, here you can view other information related to your account. From this page you can see the statistics based on your overall performance. They are divided into 4 categories, depending on the difficulty of the problems solved: all, easy, medium and hard.
    Below the role there are some links.
    If you are a student, then you have 2 buttons. One is called "class" and it takes you to the page related to the class you belong to, and the other one is called "logout", it takes you to the main page and the button Profile from the top menu will be replaced by the Login/Sign-up buttons.
    If you are a teacher, then you have 3 buttons. The logout button works the same. Instead of the "class" button, you have the "classes" button that takes you to the page related to the classes you have. In addition, you have the button "propose problem".

  6. Propose problem Page

    You can reach this page only if you are a teacher and press the "propose problem" button on the page related to your profile. Here you have a form that you can fill in with the details about the problem you want to propose. At the bottom you have 2 buttons, one for submit and one for going back. Above the submit button you have a link called "See model" which takes you to a page where you can find an already completed form.

  7. Model propose problem Page

    Here you can see a model to help you complete the problem proposal form. This page can only be accessed from the Propose problem Page.

  8. Admin Page

    This page can only be accessed by administrators. Here I can see a table with the proposed problems, and by pressing the title of a problem they can see a form completed with the details related to the problem. At the end of each line in the table, there are 2 buttons through which the administrators can approve or disapprove a proposed problem.

  9. Another user's profile Page

    You can reach this page from the viewStudents and myClass_student pages. Here you can see general information about another user as well as statistics about his/her performance.

  10. About us Page

    This page gives users some background regarding what the concept behind Informatix is and who are its creators.

  11. Help Page

    This page answers some of the most asked questions of users and for those who want to find out more, allows them to access our Scholarly in order to read more about the usage of this app.

  12. My classes Page

    This page is accessible only from a teacher account and allows the user to see their classes, to add student to these and to access information about them such as their students or their assigned homeworks.

  13. My class Page

    This page is accessible only from a student account and allows the user to see their classmates and allows them to access their profiles.

  14. View Students Page

    This page is accessible from My classes Page and allows a teacher user to view details about the students in a certain class, as well as to see the homework status for each.

  15. View Student Homeworks Page

    This page is accessible from View Students Page and allows a teacher user to view the assigned homeworks for a student, the homework status (solved yes/no), and allows the user to access View Submits Page.

  16. View Submits Page

    This page is accessible from View Student Homework Page and allows a teacher user to view the solutions a student has submited for the problems in their homeworks.

  17. View Solution Page

    This page is accessible from View Submits Page and allows a teacher user to view the code of a solution a student has submited for a problem in one of their homeworks.

  18. Add Student Page

    This page is accessible from My Classes Page and allows a teacher user to complete a form in order to add a student in one of their classes, using the username of the student.

  19. View Homeworks Page

    This page is accessible from My Classes Page and allows a teacher user to view the homeworks assigned to a certain class, as well as to access the View Problems Page. This Page also has a button for creating a new homework.

  20. View Problems Page

    This page is accessible from View Homeworks Page and allows a teacher user to view the problems that a certain homeworks is made of.

  21. View Problem Page

    This page is accessible from View Problems Page and Problems Page and allows a user to view description of a problem.

  22. Create Homework Page

    This page is accessible from View Homeworks Page and allows a teacher user to complete a form in order to create a new homework. The form allows the user to name the homework, to give it a deadline, as well as adding problems to it,based on the id of the problems.
